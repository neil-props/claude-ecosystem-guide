  <name>Task 2: Build script, HTML template, and static assets</name>
  <files>build/build.js, build/templates/page.html, docs/assets/style.css, docs/assets/main.js</files>
  <action>
1. Create `build/templates/page.html` -- the HTML shell for every generated page:
   - DOCTYPE html, lang="en", charset UTF-8, viewport meta
   - Flash-prevention script in `<head>` (same pattern as existing index.html lines 7-16): reads localStorage `claude-guide-theme`, sets `data-theme` attribute before body renders
   - `<link rel="stylesheet" href="{{basePath}}assets/style.css">`
   - `<link rel="stylesheet" href="{{basePath}}assets/hljs-theme.css">` (for highlight.js theme)
   - `<title>{{title}} - Claude Ecosystem Guide</title>`
   - Body structure: `<aside class="sidebar" id="sidebar">{{nav}}</aside>`, `<main class="content"><article>{{content}}</article></main>`
   - `<script src="{{basePath}}assets/main.js"></script>` before closing body
   - Use `{{basePath}}` placeholder for relative path resolution (empty string for root pages, `../` for topic pages)

2. Create `build/build.js` (~200-300 lines) -- the core build pipeline:
   - Import: `fs`, `path`, `glob`, `matter` (gray-matter), `marked`, `markedHighlight`, `hljs` (import from `highlight.js/lib/core` to minimize bundle)
   - Register only needed hljs languages: yaml, json, bash, javascript, typescript, markdown, xml (import each from `highlight.js/lib/languages/...`)
   - Configure marked with `markedHighlight` for build-time syntax highlighting
   - Custom marked renderer for headings: generate `id` attribute from slugified text, append anchor link `<a class="anchor" href="#slug">#</a>`
   - Custom marked renderer for code blocks: wrap in `.code-block` container with `.code-header` (language label + copy button) and `<pre><code>` inside. Use the pattern from 01-RESEARCH.md Pattern 4.
   - Read all `content/*.md` files using glob
   - For each file: parse frontmatter with gray-matter, render markdown body with marked, inject into template replacing `{{title}}`, `{{content}}`, `{{nav}}` (nav is empty string for now -- Plan 02 adds real nav), `{{basePath}}`
   - Write output: `docs/index.html` for `content/index.md`, `docs/topics/{slug}.html` for topic pages
   - Before writing: clean `docs/` directory (remove all HTML files and topics/ dir, preserve assets/ if pre-existing)
   - After rendering: copy `docs/assets/style.css` and `docs/assets/main.js` from source (these are the static assets, kept in docs/assets/ as source-of-truth for now since they're hand-crafted CSS/JS, not generated)
   - Log each page built to stdout

3. Create `docs/assets/style.css`:
   - Extract ALL CSS custom properties (variables) from existing `index.html` lines 21-68 (both dark and light theme)
   - Extract the theme transition rules (lines 70-76)
   - Add base reset (`* { margin: 0; padding: 0; box-sizing: border-box; }`)
   - Add Inter font import
   - Add body styles: `font-family: 'Inter', sans-serif; background: var(--bg); color: var(--text);`
   - Add layout grid: `.content { max-width: 800px; margin: 0 auto; padding: 2rem; }`
   - Add sidebar placeholder styles (will be enhanced in Plan 02): `.sidebar { display: none; }` for now
   - Add code block styles: `.code-block` container, `.code-header` with flex layout, `.copy-btn` button styling, `pre` and `code` styles with `var(--surface2)` background
   - Add heading anchor styles: `.anchor { opacity: 0; margin-left: 0.5rem; color: var(--text-muted); } h1:hover .anchor, h2:hover .anchor, h3:hover .anchor { opacity: 1; }`
   - Add highlight.js theme overrides that respect CSS custom properties (so hljs colors adapt to dark/light theme)

4. Create `docs/assets/main.js`:
   - Theme toggle function (extracted from existing index.html pattern): toggle `data-theme` attribute, save to localStorage, add/remove `theme-transition` class
   - System preference listener: `matchMedia('(prefers-color-scheme: light)')` change handler
   - Copy-to-clipboard: event delegation on `.copy-btn` clicks, use `navigator.clipboard.writeText(code.textContent)`, show "Copied!" feedback for 2 seconds
   - Scroll-to-anchor on page load: if `location.hash`, scroll to element with matching ID

   IMPORTANT: Do NOT use any framework or bundler. This is vanilla JS under 150 lines. Use `document.addEventListener('DOMContentLoaded', ...)` for initialization.
