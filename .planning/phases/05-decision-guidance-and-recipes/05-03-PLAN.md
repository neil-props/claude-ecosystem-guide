1. **Install Pagefind**: Run `npm install -D pagefind` to add as dev dependency.

2. **Update package.json scripts**: Add a `"build:search"` script: `"node build/build.js && npx pagefind --site docs"`. This chains the HTML build with Pagefind indexing. Keep the existing `"build"` script unchanged (just `node build/build.js`) so tests that only need HTML still work fast.

3. **Add search input to build/templates/page.html**: Insert a search container inside the sidebar, between the sidebar-header div and the sidebar-nav nav element:
   ```html
   <div class="search-container" id="searchContainer">
     <input type="search" id="searchInput" placeholder="Search docs..." aria-label="Search documentation">
     <div class="search-results" id="searchResults"></div>
   </div>
   ```

4. **Add search JS to docs/assets/main.js**: New `// ════ SEARCH ════` section. Implementation:
   - Lazy-load Pagefind module on first focus of search input (not on page load)
   - On input event (with minimum 2 chars), call `pagefindModule.debouncedSearch(query, {}, 300)`
   - If search returns null (superseded), do nothing
   - Otherwise, load first 8 results via `r.data()`, render as clickable links:
     ```html
     <a href="{url}" class="search-result">
       <div class="search-result-title">{meta.title}</div>
       <div class="search-result-excerpt">{excerpt}</div>
     </a>
     ```
   - Clear results when input is empty or less than 2 chars
   - Close results when clicking outside (add click listener on document)
   - Handle basePath: Pagefind returns absolute paths like `/topics/mcp.html`. For GitHub Pages deployment under a subdirectory, the paths should work as-is since the site is at root. But use relative paths by stripping leading `/` if the URL starts with `/topics/` or `/index`.
   - Graceful degradation: wrap Pagefind import in try/catch. If pagefind.js doesn't exist (hasn't been indexed yet), show a small "Search not available" message instead of breaking.

5. **Add search CSS to docs/assets/style.css**:
   - `.search-container`: padding 0.75rem 1rem, position relative, border-bottom using `var(--border)`
   - `#searchInput`: full width, padding 0.5rem 0.75rem, border 1px solid `var(--border)`, border-radius `var(--radius-sm)`, background `var(--surface)`, color `var(--text-primary)`, font-family Inter, font-size 0.875rem. Focus state: border-color turquoise, outline none, box-shadow subtle
   - `.search-results`: position absolute, top 100%, left 1rem, right 1rem, background `var(--surface-raised)`, border 1px solid `var(--border)`, border-radius `var(--radius-sm)`, max-height 400px, overflow-y auto, z-index 100, box-shadow `var(--shadow-lg)`
   - `.search-result`: display block, padding 0.75rem 1rem, border-bottom 1px solid `var(--border)`, text-decoration none, color `var(--text-primary)`. Hover: background `var(--surface-hover)`.
   - `.search-result-title`: font-weight 600, font-size 0.875rem, margin-bottom 0.25rem
   - `.search-result-excerpt`: font-size 0.8rem, color `var(--text-secondary)`, line-height 1.4. Style `mark` tags inside excerpt: background `var(--accent-turquoise)` with 0.2 opacity, padding 0 2px, border-radius 2px
   - `.search-results:empty`: display none

6. **Create tests/test-search.js**: After build (HTML build only, not Pagefind indexing), verify:
   - Template output contains `id="searchInput"` element

7. **Run Pagefind indexing**: After all the above, run `npx pagefind --site docs` to generate the search index. The `docs/pagefind/` directory should be created with index files. Verify the directory exists and contains `pagefind.js`.
