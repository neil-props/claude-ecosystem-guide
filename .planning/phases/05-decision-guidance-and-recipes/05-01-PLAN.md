5. **Add wizard state machine to docs/assets/main.js** in a new `// ════ DECISION WIZARD ════` section after the tab switching section. Implement:
   - `wizardSteps` object with these decision nodes:
     - `start`: "What are you trying to do?" → 5 options (connect external tool/API, teach repeatable process, run code automatically, configure project behavior, create specialized workflow)
     - `external-tool`: "What kind of integration?" → build/host own server (→ mcp), pre-built one-click (→ connectors), bundle as distributable package (→ plugins)
     - `repeatable`: "How complex is the process?" → simple instructions/checklist (→ skills), multi-step with tool access (→ agents), quick prompt shortcut (→ commands)
     - `automation`: "What triggers the automation?" → code lifecycle events (→ hooks), scheduled/on-demand (→ agents), project-specific rules (→ skills)
     - `configure`: "What level of configuration?" → project-wide instructions (→ projects), behavior settings (→ settings), persistent context (→ memory)
     - `specialized`: "What kind of workflow?" → domain expert with specific knowledge (→ skills), autonomous multi-step agent (→ agents)
   - Each result node: has `result` key (slug name), display name, one-sentence description, and link to the topic page (`topics/{slug}.html`)
   - `renderWizardStep(stepId)` function: clears wizard container, renders question as h3, renders options as buttons with class `wizard-option`, attaches click handlers
   - `renderWizardResult(resultId)` function: shows recommendation card with extension point name, description, link to topic page, and a "Start Over" button
   - History array for back navigation: push stepId on each forward step, "Back" button pops and re-renders
   - URL hash state: update `location.hash` to `#step={stepId}` on navigation, read hash on page load to restore state
   - Initialize wizard only if `#wizard` element exists on page

   - `.wizard-option`: full-width button, left-aligned text, border using `var(--border)`, padding 1rem 1.25rem, hover state with turquoise accent border-left (4px), cursor pointer, margin-bottom 0.5rem, background `var(--surface-raised)`, border-radius `var(--radius-sm)`

   - `.wizard-result-link`: styled as primary button/link
   - `.wizard-back`: subtle link-style button, positioned above options
   - `.wizard-progress`: simple breadcrumb showing path taken (e.g., "Start > External Tool")
