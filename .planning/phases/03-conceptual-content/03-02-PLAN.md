**memory.md (CONC-09):**
- Enhance Overview to explain memory as "how Claude remembers things across sessions" -- the system for persisting context beyond a single conversation
- Explain the 4-layer architecture clearly:
  1. **CLAUDE.md files** -- Manual project instructions (always loaded)
  2. **Auto-memory** (~/.claude/memory/) -- Claude automatically saves learnings from conversations
  3. **/memory command** -- User explicitly tells Claude to remember something
  4. **Agent memory** -- Agents can have their own persistent context
- Add "When to Use Each Layer" decision guidance:
  - CLAUDE.md: project standards, coding conventions, workflow instructions
  - Auto-memory: let Claude learn your preferences organically
  - /memory: important one-off facts ("my API key format is X", "always use tabs not spaces")
  - Agent memory: specialized knowledge for custom agents
- Cover persistence: what survives session restarts, what is project-scoped vs global

**settings.md (CONC-10):**
- Enhance Overview to explain settings as "the configuration hierarchy that controls Claude Code's behavior" -- how different scopes of settings merge and override each other
- Explain the 5-level precedence clearly (highest to lowest):
  1. **Managed policy** (enterprise/organization)
  2. **CLI flags** (per-invocation)
  3. **settings.local.json** (per-project, gitignored)
  4. **settings.json** (per-project, committed)
  5. **User settings** (~/.claude/settings.json)
- Add "Why This Matters" narrative: settings hierarchy lets teams enforce standards while individuals customize their experience
- Add "When to Use Each Level":
  - Managed: enterprise security policies (admins only)
  - CLI flags: temporary overrides for specific tasks
  - settings.local.json: personal project preferences (API keys, model preferences)
  - settings.json: shared team settings (approved tools, MCP servers)
  - User settings: global personal defaults
