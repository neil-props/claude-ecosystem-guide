Three Interfaces

Same Claude brain, three different surfaces optimized for different users and workflows.

⌨

Claude Code

For Developers

An agentic CLI for your terminal. Reads your entire codebase, edits multi-file projects, runs commands, creates commits, opens PRs, and ships code autonomously.

Available in Terminal, VS Code, JetBrains, GitHub Actions, GitLab CI
Full filesystem Git integration Bash execution Parallel subagents Worktree isolation Extended thinking
💬

Claude Chat

For Everyone

The conversational web & desktop app at claude.ai. Upload files, analyze images, search the web, use interactive apps, and work within persistent Projects.

Available at claude.ai (web), macOS, Windows, iOS, Android
File uploads Web search Interactive Apps Connectors (50+) Projects Artifacts
🛠

Claude Cowork

For Knowledge Workers

A desktop agent for non-developers. Describe an outcome, step away, and return to finished work — organized files, reports, spreadsheets, presentations.

Launched Jan 2026 · macOS & Windows · Pro/Max/Team/Enterprise
Folder access Coordinator + sub-agents 11+ plugins Scheduled tasks Slash commands No code required

The Customization Stack

Three layers that build on each other. MCP connects to services. Skills teach processes. Plugins package everything for distribution.

Product Layer
Plugins
Complete, installable workflow packages. Bundle skills + connectors + slash commands + sub-agents. One-click install from a marketplace.
Code Chat Cowork
Instruction Layer
Skills
Portable instruction sets (SKILL.md files) that teach Claude specific processes. Same format works everywhere. Open standard at agentskills.io.
Code Chat Cowork
Transport Layer
MCP Servers & Connectors
Open protocol (JSON-RPC 2.0) connecting Claude to live services — databases, APIs, SaaS apps. The "USB-C" of AI connectivity.
Code Chat Cowork
Skills are the universal primitive

A skill is a folder with a SKILL.md file. The exact same skill works in Claude Code, Claude Chat, and Cowork without modification. Build once, use everywhere. Launched Oct 2025 as an open standard.

Plugins are the distribution unit

Plugins bundle multiple skills, connectors, slash commands, and sub-agents into a single installable package. 11+ official plugins (Sales, Finance, Legal, etc.) plus the ability to build your own.

All Extension Points

Seven ways to customize Claude. Click any card for details.

MCP Servers / Connectors

Connect Claude to external tools & APIs. Called "Connectors" in Chat (50+), configured via CLI or .mcp.json in Code. Donated to Linux Foundation Dec 2025.
Code Chat Cowork

Skills

Portable instruction sets (SKILL.md). Claude auto-invokes them based on task context via progressive disclosure. Same open format across all surfaces.
Code Chat Cowork

Plugins

Installable bundles of skills + connectors + agents + hooks + slash commands. 15+ official plugins with 85+ skills. Marketplace distribution.
Code Chat Cowork

Projects / CLAUDE.md

Persistent context. CLAUDE.md files in Code (auto-loaded per repo). Named project spaces with knowledge bases in Chat.
Code Chat Cowork

Subagents / Agent Teams

Specialized AI assistants with isolated context, custom prompts, tool restrictions, and their own models. Run in parallel or in git worktree isolation.
Code Chat Cowork

Hooks

Event-driven automation. Run scripts, call APIs, or invoke Claude at 16+ lifecycle points (PreToolUse, SessionStart, etc.).
Code Chat Cowork

Interactive Apps

Live app interfaces (Slack, Figma, Asana, Canva) embedded directly inside Claude Chat conversations via the MCP Apps extension.
Code Chat Cowork

Which Should I Use?

Common scenarios mapped to the right interface and extension.

"I want Claude to talk to Slack / Asana / our database"
Connect Claude to external services using MCP Servers (in Code) or Connectors (in Chat). Both use the same protocol. 50+ pre-built connectors available in Chat; thousands of community MCP servers for Code.
MCP Server / Connector
"I want Claude to generate Excel reports in a specific format"
Create a Skill with your report template and formatting instructions. Skills work across all three interfaces — the same SKILL.md file will work whether your team uses Chat, Code, or Cowork.
Skill
"I want a reusable /deploy command for my dev team"
Create a Skill at .claude/skills/deploy/SKILL.md and check it into your repo. For distribution across multiple repos, bundle it in a Plugin and host on GitHub.
Skill or Plugin
"Claude should always know our coding conventions"
In Code: put conventions in CLAUDE.md at your repo root (auto-loaded every session). In Chat: create a Project with custom instructions. In Cowork: use folder-specific instructions.
Project / CLAUDE.md
"I want to auto-lint code before Claude commits"
Set up a Hook on PreToolUse or PostToolUse events. Hooks run shell scripts, call APIs, or invoke Claude at 16+ lifecycle points. Claude Code only.
Hook Code only
"Our finance team wants Claude to help with reconciliation"
Install the Finance Plugin in Cowork from the Anthropic marketplace. One-click install, includes specialized skills for journal entries, reconciliation, variance analysis. For Chat users, the same skills are available individually.
Plugin (Cowork) or Skills (Chat)
"I need Claude to process 200 invoices while I'm away"
Claude Cowork is built for autonomous file tasks. Grant it folder access, describe the task, and step away. It uses a coordinator + sub-agent model to parallelize work. You can even schedule recurring tasks.
Cowork
"I want to share a full toolkit across my engineering org"
Build a Plugin that bundles your team's skills, subagents, hooks, and MCP configs. Host on a GitHub marketplace repo. For org-wide enforcement, use managed settings and managed-mcp.json.
Plugin
"I want to use Figma/Slack inside a Claude conversation"
Interactive Apps in Claude Chat embed live app interfaces directly in your conversation. Built on the MCP Apps extension. Currently available for Slack, Figma, Asana, Canva, Box, Clay, and more.
Interactive Apps (Chat)
